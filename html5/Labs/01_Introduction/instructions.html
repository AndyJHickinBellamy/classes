Lab 01 - Introduction (and a contest!)

Overview

In this lab we'll create a simple HTML 5 page. You can use the HTML editor of your choice - Visual Studio, Notepad, TextMate, or vi. You also use any browser you like. Create your files in the before directory of this lab. You can find a completed solution in the after directory.

The goal is to create a newspaper / new portal page using news you find on the web (or make up from your imagination). Submit your entries to the instructor's shared drive and the class will vote on the best page! You have 50 minutes to follow the instructions. Once you've completed the instructions, use your spare time to make your page beautiful!

Creating the page

Create a new HTML page - page.html

  1. Give the page an HTML 5 doctype
  2. Set the language of the page to english (or your preferred language). The language is specified on the html element
  3. Add a head tag to the page with a title element inside. Pick your own title.
  4. Using a meta tag, set the character encoding for the page to utf-8.
  5. Add an empty body tag to the page.

The page should look similar to the following. 

<!doctype html>
<html lang="en">
<head>
    <meta charset="utf-8">
    <title>My first lab</title>
</head>
<body>

</body>
</html>

Adding some content

In this section we want to create an online newspaper using HTML 5 semantic markup.

  1. Pick up some articles from your favorite news site (perhaps Reuters), or make up your own news.
  2. Use the news content as the content for your page. You should be able to use the following elements to add meaning to the page.
    • section
    • article
    • summary
    • hgroup
  3. If you need help understanding what an element means, feel free to look at articles on the web, or refer to the W3C
  4. Find one or more pictures for your news article(s) and add them using figure and figcaption elements.

The result might look something like the following (the following would be inside the body elements, of course). 

<section>
    <article>
        <header>
            <hgroup>
                <h1>Millions of escaped bees shut down highway</h1>
                <h2>Truck overturns</h2>
            </hgroup>
            <time datetime="2010-10-25">Oct 25th, 2010</time>
        </header>
        <p>
            Local beekeepers worked through the night to gather the escaped bees.
            Officials said there was a net over the beehives but bees still managed
            to escape after the truck overturned.
        </p>
    </article>
</section>

Validation

  1. Take your source code to the W3C Validator
  2. Click on "Validate By Direct Input"
  3. Paste your code into the validator, click "Check", and review the results
  4. The markup shown in the last section will generate a validation error. If your code doesn't have an error, use the markup from the previous section to see what an error looks like (hint: one of the elements should be changed).
  5. Ensure the source code passes validation checks as HTML 5

Adding Some Style

  1. Create an new CSS file (you might name it styles.css)
  2. Add a link in your html page to use the style sheet. Remember in HTML 5 you don't need a type attribute.
  3. Set the style for the body of your page to have a light grey background (#cccccc), or some other color of your preference.
  4. Use caniuse.com to determine if your browser supports a box shadow, and if it requires a vendor prefix (Ttping in the search box will take you directly to a feature).
  5. Add a box shadow to the figures in your page. Refer to the W3C spec section 7.2 for help.
  6. Experiment with different shadow lengths and using the inset keyword to create different effects. You can also use your favorite search engine to find interactive CSS box shadow generators.
  7. Once the shadow looks good (and square), find out if your browser supports a border radius, and make the shadow curve.

The result might look like the following. 

body{
    background-color: #cccccc;
}

article {
    box-shadow: 15px 15px 15px  black;
    border-radius: 15px;
}
